Theory

Understanding the fundamentals of Syntax Analysis

What is a Syntax Analyser?

A Syntax Analyser (also called a Parser) is the second phase of a compiler. It takes the stream of tokens produced by the Lexical Analyser and checks whether that token sequence can be generated by the grammar of the source language. It organizes tokens into a Parse Tree (Syntax Tree) that reflects the structure of the program.


Key Functions

  • Receives a token stream from the Lexical Analyser
  • Validates the token sequence against a Context-Free Grammar (CFG)
  • Builds a Parse Tree / Abstract Syntax Tree (AST)
  • Reports syntax errors with line numbers and expected tokens
  • Applies derivation rules (top-down or bottom-up)
  • Handles operator precedence and associativity

How It Works

The parser reads tokens one by one and tries to match them against the production rules of the grammar. For example, given the grammar rule E → E + T | T, the parser checks if the input can be derived from the start symbol. If the input matches the grammar, a parse tree is built; otherwise, a syntax error is reported.


Example

For the expression a + b * c, the parser produces:

        E
       / \
      E   T
      |  /|\
      T T * T
      |  |   |
      a  b   c

This tree respects operator precedence — multiplication binds tighter than addition.

Grammar Rules

Sample Context-Free Grammar used in the Syntax Analyser

# Production Rule Description
1 E → E + T Expression is an Expression plus a Term (addition)
2 E → E - T Expression is an Expression minus a Term (subtraction)
3 E → T Expression can be a single Term
4 T → T * F Term is a Term times a Factor (multiplication)
5 T → T / F Term is a Term divided by a Factor (division)
6 T → F Term can be a single Factor
7 F → ( E ) Factor is a parenthesized Expression
8 F → id Factor is an identifier (variable name)
9 F → num Factor is a numeric constant

Parsing Types

Top-Down vs Bottom-Up parsing strategies

Feature Top-Down Parsing Bottom-Up Parsing
Direction Start symbol → Input string Input string → Start symbol
Tree Construction Root to Leaves Leaves to Root
Derivation Leftmost derivation Rightmost derivation (in reverse)
Examples Recursive Descent, LL(1) Shift-Reduce, LR(0), SLR, LALR
Used In Hand-written parsers Parser generators (yacc, bison)
Power Less powerful (LL grammars) More powerful (LR grammars)
